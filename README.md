🛸 Проект: ForVide — Платформа Видеохостинга нового поколения
1. Концепция и Видение
ForVide — это современная видео-платформа, сочетающая эстетику Web 3.0, "Bento Grid" дизайна и высокую функциональность. Сайт должен выглядеть дорого, технологично и быть максимально удобным для контент-мейкеров.

Ключевой стиль: "Технологичный минимализм" с агрессивными дизайнерскими акцентами.

Цветовая палитра: Горчично-желтый (#facc15), глубокий черный (#0a0a0a), чистый белый.

Логотип: Расположен в main/icons/global.png. Должен использоваться как основной элемент идентификации.

2. Дизайн-система (Для Дизайнеров)
UI/UX Элементы:
"Желтая Капля" (Боковое меню): * Это уникальный идентификатор интерфейса.

Форма: Асимметричная каплевидная форма (скругление только с левой стороны).

Функционал: Обязательная кнопка «Закрыть» (крестик) и иконки для каждого пункта меню (использовать Phosphor Icons).

Состояние: Плавный выезд справа с эффектом backdrop-filter: blur.

Bento Grid & Карточки:

Сетка видео должна быть адаптивной.

Карточки: Острые углы (clip-path на превью), мягкие, но глубокие тени.

Инфо-блок: Под каждым видео: аватар автора, никнейм, количество подписчиков, просмотры, лайки/дизлайки и таймкод длительности.

Эффекты:

Использование теней и градиентов для создания объема.

Hover-анимации на всех интерактивных элементах (масштабирование, изменение цвета).

3. Функциональные требования (Для Программистов)
Ядро системы:
Стек: HTML5, CSS3 (модульный подход), JavaScript (ES6+), Firebase.

Firebase Integration:

Auth: Вход через Email и Google Account.

Firestore: Хранение данных о пользователях, видео и комментариях.

Storage: Загрузка тяжелых медиафайлов и превью.

Система ролей:
Зритель (Default): Просмотр, лайки, комментарии. Кнопки «Загрузить» и «Стримить» заблокированы.

Автор: Появляется после прохождения флоу «Создать канал» в VideStudio. Получает доступ к загрузке.

Модули:
VideStudio (Творческая студия): Отдельное пространство для авторов с аналитикой и формой загрузки.

Алгоритм "Агент": Скрипт, который анализирует теги загруженных видео и динамически формирует "Тренды" с забавными названиями (например: "ИГРАЕМ В..", "ЛЕГЕНДЫ?", "ОБЩИЙ СБОР!").

Поиск: Глобальный поиск по названиям и авторам с дебаунсом (оптимизация запросов).

Плеер: Уникальные динамические страницы для каждого видео (player.html?id=...).

4. Архитектура файлов
Проект должен быть строго модульным для легкой поддержки:

/css — разделение на global.css, layout.css, components.css, studio.css.

/js — разделение на auth.js, api.js (Firebase), ui-controller.js, player-logic.js.

/main/icons — хранилище графических активов.

5. Гайд для Бета-тестеров (QA)
На что обратить внимание при тестах:

Флоу регистрации: Корректность создания записи в Firestore при первом входе.

Загрузка видео: Проверка прогресс-бара, валидация форматов файлов (только видео и изображения).

UX Капли: Удобство закрытия меню на мобильных устройствах.

Роли: Убедиться, что "Зритель" не может получить доступ к studio.html через прямой URL (редирект).

Скролл: Плавность подгрузки новых секций при прокрутке вниз.

6. Технический стек (Summary)
Frontend: Vanilla JS / HTML / CSS.

Backend/DB: Firebase (Auth, Firestore, Storage).

Icons: Phosphor Icons.

Fonts: Montserrat (400, 600, 800, 900).
